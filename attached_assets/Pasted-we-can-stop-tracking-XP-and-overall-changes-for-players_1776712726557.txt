/actions (member level command) > opens menu that displays the following buttons: 

ROW 1: 
Make A Purchase; Place A Wager; View/Transfer/Send Coins; Request an Interview; Post A Tweet;

View My Roster; View Any Roster; View Free Agents; View Player Stats & Ratings; View Team Stats;

ROW 1: 
Make A Purchase > opens new menu with three buttons on Row 1: Attribute Upgrade; Age Reset; Dev Trait Upgrade; buttons on Row 2: Custom Player; Legend; and a close button at the bottom of the message that will return the user to the main menu. If any of these features are turned off in the server settings, then their button should not display. If all store functions - legends, custom players, attribute, dev trait upgrades and age resets - are turned off, then the "Make a Purchase" button shouldn't even show when /actions is ran by a user. If even one of these functions is turned on for the server, the button should show. This same logic should apply for any button whose settings are turned off in server settings. Which ever option is selected should progress to the next menu where each option should function as their current commands do, ie Attribute upgrade should take the user through a menu selection of what attribute(s) their upgrading, increase amount via dropdown, capped at whatever remaining points they have for core/non-core for the season, etc. Same with age resets and dev trait upgrades. Legend button should display the list of available legends in store by position group and then require the user select a position group via dropdown and then a legend from a dropdown list populated only by available legends in that position group. Custom player button should take the user to the custom player archetypes view, where they can choose a position and view the related archetypes for that position. should have a "Purchase and Customize" button that confirms an archetype and begins the purchase and creation process just as our current command for buying custom players does. 
Place A Wager > opens a menu that functions the same as the /wager command. Allows the user to select a game from the current week from a drop down and then select one of the two teams as their pick to win. they then enter the coin amount for the wager click "Proceed" button > takes the user to another menu where they'll select a user from the AFC/NFC dropdowns who is actively linked to a team in the server. Then will click "Send Wager" button to send it as usual, or click Cancel to return to menu. Should function same as /wager command does. 
View/Transfer/Send Coins > opens a menu with three buttons on Row 1: View Coin Balance (shows the users in-server wallet, global savings, and global funds from all server wallets); Transfer Coins (allows the user to set an amount to transfer coins from their in-server wallet to their global savings or select to transfer FROM global savings to their in-server wallet); Send Coins (allows user to send a specified coin amount to another user via selecting them from an AFC/NFC dropdown of linked users, cannot go negative). Also has a cancel button that returns to main menu. 
Request an Interview > opens a menu with three buttons on Row 1: Pre-Game Interview; Post-Game Interview; General Interview; If pre-game is clicked, the new menu will display the users opponent for the current week and provide them with three AI generated questions related to this opponent and their upcoming matchup. The AI will use all guild specific data related to the two teams, to include season stats, recent performances, w/l record to generate the three questions that the user can respond to in text fields. Once submitted, posts to the league headlines channel just as the interviewrequest command does and issues a payout to the user. If Post-Game Interview is selected, will display the users current opponent for the week and ask them to input the final game score for each team. It will then use this data as well as any other relevant guild-specific team data and season data to formulate 3 AI generated questions regarding the game for the user to answer. Same end process of payout being issued. General Interview button opens a menu prompting the user to input the topic for their requested interview. AI will then pull all relevant league-specific data and metrics to form 3 AI generated questions related to the topic the user submitted and allow them to answer for a payout. Users are still limited to one interview per in-game week, so if a user has already submitted theirs for the week then this button will not appear again that week when /actions command is run by a user. will be available again upon advance to a new week. 
Post A Tweet > opens a menu where it will prompt the user to input their tweet text and submit. once submitted, it will post an embed to the league twitter channel with their @nickname and the tweet itself, shared by the bot and tagging @everyone in the tweet. A user is limited to 2 tweets per week and the tweet payouts are 5 coins each by default.

You'll need to add the option to edit tweet and interview payouts in the /admin-payout command.
